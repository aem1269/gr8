Student User Guide

As a student of a university using the Gr8ing SystemTM, you can perform the following actions:

  • Add courses to your shopping cart
  • Enroll in courses
  • View your schedule graphically on a calendar
  • Check conflicts between courses in your shopping cart and your Google Calendar.
  • Export your course schedule to Google Calendar
  • View your grades for previous courses
  • Generate an unofficial transcript from your course history

Shopping Cart

In order to add a course to your shopping cart, navigate to the Current Courses page, click on the course you would like to add, and click the 'Add to Cart' button. Once a course has been added to the shopping cart, it can be removed by navigating to the shopping cart, clicking on the course, and pressing the 'Remove From Cart' button. You can also check the courses in your shopping cart for conflicts with your Google Calendar on the Shopping Cart page by clicking the 'Check Conflicts Against Google Calendar' button.

Enrollment

To enroll in a course, navigate to a course offered during the desired semester, then click enroll. Alternativly, you can add courses to your shopping cart, then enroll in all of them at the same time by clicking the 'Enroll All' button on the Shopping Cart page. To drop a course, simply navigate to the page of the course you wish to drop, then click the 'Drop' button.

Schedule

To view your schedule, navigate to the 'My Schedule' page. On the schedule page, you can click on the course block in the schedule to go to the info page for a course. Additionally, you can post your course schedule to Google Calendar by clicking the button at the bottom of the page.

Grades

To view your grades, go to the 'My Grades' page. This page will show grades for every course you have taken. You can also view an unofficial transcript by clicking the 'Generate Transcript' button at the bottom of the page.

Professor User Guide

Professors in the gr8ing system are profiles that have been assigned to teach a course. The only extra functionality that a professor has over a student is that when a professor goes to the course information page for a course they are teaching they see an enter grades button (if the course has at least one student). When clicked a list of students that are enrolled in the course are shown with an input field for their grade, the grades can be saved, and then modified again if necessary. Professors may enroll in any class they want to and they will be treated like a student.

Admin User Guide

As and Admin of a university using the Gr8ing SystemTM, you are given privileged access to functionality that the system depends on to exist. Admins can access the Admin dropdown in the header when logged in. Admins are responsible for tasks such as

  • Creating Users
  • Creating Rooms
  • Creating Terms
  • Creating Creating Course Codes
  • Creating Courses

Creating Users

Whenever a new user needs to be added to the system, an admin must must create an account for the new user. The admin enters the users personal information and creates a user name and password for the account. The admin also has to set if the user can enroll in classes, if the checkbox is not checked the user cannot enroll in classes, they can just view them and add them to their shopping cart. For example if there was a financial hold on the student you would not want them to be able to enroll until the finical hold was resolved.

Creating Rooms

For classes to actually be held there has to be rooms to hold them in. Admins must add every room in which classes are taught to the system. This will be a big effort when the system is new, but once all the rooms are entered, they won't have to be entered again. After all the rooms have been entered, the create a room page will not be needed unless a new room is added to the university and it needs to be added to the gr8ing system.

Creating Terms

For the University to be able to hold classes in an organized fashion, the define periods of time known as terms. These terms have to be entered into the gr8ing system by the Admins so that courses can have a term they belong to. Terms are created by selecting the season and then setting a start and end date for the term.

Creating Course Codes

For courses to exist there first has to be course codes in the system. Course codes have a code such as "SWEN-261" or "MATH-182" these codes are used as an abbreviation for the name of the course which can get quite long. When creating a course code the admin also has to set the number of credits the course is worth, and they can add any pre-requisites the course code has, for example "SWEN-261" is a pre-req of "SWEN-262". A course can have between 0 and 3 pre-reqs so the fields can be left blank.

Creating Courses

Once the Admins have created course codes, terms and rooms, courses can be created. Each course can have 0 to N sections and the gr8ing system auto increments the section number as courses with the same course code are created. The professor for the class is set to a user in the system, or it can be left blank to leave the professor set as "Staff" if the professor for the class has not yet be decided. When creating the course the Admin creates course sessions, which define where and when the course meets. This is done by selecting the days of the week, the time (in military time) and the room the course will meet. There can be as many course sessions as desired, allowing the course to meet on different days at different times.
